# 新订单流程 - 部署和测试指南

## 📦 修改内容总结

### 1. 云函数修改

#### `cloudfunctions/order/index.js`

**completeOrder 函数**（上传照片）：
- 修改前：上传后状态变为 `completed`
- 修改后：上传后保持 `shipped` 状态

**confirmReceipt 函数**（确认收款）：
- 修改前：检查 `completed` 状态，完成后变为 `closed`
- 修改后：检查 `shipped` 状态，完成后变为 `completed`

### 2. 前端修改

#### `pages/craftsman/orderDetail.wxml`
- 更新状态标签颜色（已发货：橙色，已撤回：红色）
- 修改按钮逻辑：
  - `shipped` + 未传照片 → "完成订单（上传照片）"
  - `shipped` + 已传照片 → "确认收款"
  - `completed` → "订单已完成"
  - `closed` → "订单已撤回"

#### `pages/craftsman/orderDetail.js`
- 更新 `checkCanConfirmReceipt`：检查 `shipped` 状态

#### `pages/craftsman/myOrders.wxml/js`
- 筛选选项："已结束" → "已撤回"
- 列表按钮逻辑更新
- 状态标签更新

#### `pages/craftsman/profile.wxml`
- 添加"已发货"统计入口

## 🚀 部署步骤

### 步骤 1: 部署云函数

在微信开发者工具中：

1. 找到 `cloudfunctions/order` 文件夹
2. 右键点击 → "创建并部署：云端安装依赖"
3. 等待部署完成

或者使用命令行：
```bash
cd cloudfunctions/order
wxcloud deploy --env <你的环境ID>
```

### 步骤 2: 清除缓存

在微信开发者工具中：
1. 点击 "清除缓存" → "全部清除"
2. 重新编译项目

## 🔍 问题排查

### 问题 1：确认收款按钮不显示
**检查点**：
1. 订单状态是否为 `shipped`
2. 是否已上传照片（`completePhotos` 数组是否有值）
3. 当前用户是否为接单人

**调试方法**：
查看页面顶部调试信息区域（黄色背景），检查：
- `status: shipped`
- `completePhotos` 是否有值

### 问题 2：上传照片后状态变了
**检查**：云函数是否已正确部署
- 重新部署 `order` 云函数
- 清除微信开发者工具缓存

### 问题 3：状态显示不正确
**检查**：WXML 中的状态判断条件
- `shipped` 状态显示"已发货"+"待收款"
- `completed` 状态显示"已完成"+"已收款"
- `closed` 状态显示"已撤回"

| 最终状态 | closed | completed |
