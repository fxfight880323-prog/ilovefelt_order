# 订阅消息配置说明

## 错误原因

错误 `errCode: 20001` 表示订阅消息模板ID未配置或无效。

## 解决方法

### 1. 申请订阅消息模板

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入 "功能" → "订阅消息"
3. 点击 "添加" 或 "从模板库中选择"
4. 搜索以下关键词找到合适的模板：

| 用途 | 搜索关键词 |
|------|-----------|
| 新订单提醒 | 订单、新订单、接单 |
| 订单状态变更 | 订单状态、订单变更 |
| 订单完成 | 订单完成、完成通知 |
| 审核结果 | 审核、审核结果 |

5. 选择合适的模板，点击 "选用"
6. 配置模板关键词（按需选择）
7. 提交后获取模板ID（格式类似：`x1ixxxxxxxxxxxxxxxxxxxxx`）

### 2. 配置模板ID

打开 `config/subscribe.js` 文件，将获取的模板ID填入：

```javascript
module.exports = {
  // 新订单提醒
  NEW_ORDER: '你的模板ID1',
  
  // 订单状态变更提醒
  ORDER_STATUS_CHANGE: '你的模板ID2',
  
  // 订单完成提醒
  ORDER_COMPLETE: '你的模板ID3',
  
  // 审核结果通知
  AUDIT_RESULT: '你的模板ID4'
}
```

### 3. 重新编译

在微信开发者工具中：
1. 点击 "编译"（Ctrl+R）
2. 清除缓存（工具 → 清除缓存 → 全部清除）
3. 重新测试

## 模板推荐

如果没有合适的模板，可以申请以下类型的模板：

### 1. 新订单提醒
- **类目**：生活服务 / 电商平台
- **关键词**：订单类型、金额、时间
- **示例内容**：您有新的{{thing1.DATA}}订单，金额{{amount2.DATA}}元，请尽快处理

### 2. 订单状态变更
- **类目**：生活服务
- **关键词**：订单状态、处理结果
- **示例内容**：您的订单{{thing1.DATA}}状态已变更为{{thing2.DATA}}

### 3. 审核结果通知
- **类目**：生活服务
- **关键词**：审核结果、原因
- **示例内容**：您的{{thing1.DATA}}审核{{phrase2.DATA}}，原因：{{thing3.DATA}}

## 注意事项

1. **模板审核**：自定义模板需要1-3个工作日审核
2. **类目限制**：不同类目有不同模板可选
3. **使用限制**：每个用户订阅次数有限，需要用户主动订阅
4. **长期订阅**：部分模板可申请长期订阅（无需用户每次确认）

## 临时解决方案

如果不想配置订阅消息，可以直接忽略该错误，系统会自动跳过订阅功能，不影响其他功能使用。
