# 订阅消息只弹一次功能测试指南

## 📋 功能说明

- 用户第一次进入接单大厅时，弹出订阅提醒
- 用户接受或拒绝后，记录到数据库
- 之后不再重复提示（即使拒绝也不再打扰）

---

## 🧪 完整测试步骤

### 1. 准备工作

```bash
# 1. 部署 craftsman 云函数（重要）
右键 cloudfunctions/craftsman → 创建并部署：云端安装依赖

# 2. 清除缓存测试
微信开发者工具 → 工具 → 清除缓存 → 全部清除

# 3. 数据库初始化（可选）
# 如果 craftsmen 表中没有 subscribeChoice 字段，会自动添加
```

### 2. 首次进入测试

**步骤：**
1. 清除小程序缓存（模拟新用户）
2. 手艺人登录，进入"接单大厅"

**预期结果：**
- 页面加载1秒后弹出"开启消息提醒"弹窗
- 控制台输出：`用户未做过订阅选择，显示提示`
- 显示"🔔 开启消息提醒"提示卡片

### 3. 用户接受订阅测试

**步骤：**
1. 点击"去开启"
2. 勾选"新订单提醒"
3. 点击"允许"

**预期结果：**
- 提示"订阅成功"
- 提示卡片消失
- 控制台输出：`保存订阅选择到数据库: accepted`
- 数据库 craftsmen 表中该用户记录新增字段：
  ```json
  {
    "subscribeChoice": "accepted",
    "subscribeChoiceTime": "2024-01-15T10:30:00.000Z"
  }
  ```

### 4. 再次进入测试（已接受）

**步骤：**
1. 退出接单大厅
2. 重新进入接单大厅

**预期结果：**
- **不弹出订阅弹窗**
- **不显示提示卡片**
- 控制台输出：`用户已订阅，不显示提示`

### 5. 清除缓存后再次进入测试

**步骤：**
1. 清除小程序缓存
2. 重新进入接单大厅

**预期结果：**
- 从数据库读取到已接受记录
- 控制台输出：`数据库中的订阅选择: accepted`
- **不弹出订阅弹窗**

### 6. 用户拒绝订阅测试（新用户）

**步骤：**
1. 使用新手艺人账号（数据库中无 subscribeChoice 记录）
2. 进入接单大厅

**步骤：**
1. 点击"去开启"
2. 点击"取消"或关闭弹窗

**预期结果：**
- 提示"已取消订阅提醒"或"订阅已取消"
- 提示卡片消失
- 控制台输出：`保存订阅选择到数据库: rejected`
- 数据库记录 subscribeChoice 为 rejected

### 7. 拒绝后不再打扰测试

**步骤：**
1. 被拒绝过的手艺人重新进入接单大厅

**预期结果：**
- 控制台输出：`数据库中的订阅选择: rejected`
- 控制台输出：`用户之前拒绝过订阅，不再提示`
- **不弹出订阅弹窗**
- **不显示提示卡片**

## ⚠️ 常见问题

### 问题1：每次进入都弹窗
**原因**：数据库中没有保存选择记录
**检查**：
- 云函数是否正确部署
- 数据库权限是否允许更新
- 控制台是否有保存失败的错误

### 问题2：拒绝了还弹窗
**原因**：没有从数据库读取选择记录
**检查**：
- checkSubscribe 函数是否正确执行
- getSubscribeChoice 接口是否返回正确数据

### 问题3：接受订阅后还显示提示卡片
**原因**：微信设置和数据库状态不同步
**检查**：
- 检查微信设置中是否显示已订阅
- 数据库中 subscribeChoice 是否为 accepted
