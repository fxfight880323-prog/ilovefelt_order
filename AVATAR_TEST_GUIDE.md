# 头像上传功能测试指南

## 📋 测试前准备

1. **部署云函数**
   ```
   右键 cloudfunctions/user → 创建并部署：云端安装依赖
   ```

2. **重新编译小程序**
   ```
   按 Ctrl + R 重新编译
   ```

---

## 🧪 测试步骤

### 测试1：上传头像

1. **进入个人中心**
   - 切换到手艺人角色
   - 点击"我的"进入个人中心

2. **点击头像**
   - 点击当前头像（或默认头像）
   - 选择"从相册选择"或"拍照"

3. **预期结果**
   - 显示"上传中..."
   - 提示"上传成功"
   - 头像立即更新为新图片

### 测试2：刷新后头像保留

1. **下拉刷新页面**
   - 在个人中心下拉刷新

2. **重新进入页面**
   - 切换到其他页面再回来

3. **预期结果**
   - 头像仍然显示已上传的图片
   - 不是默认头像

### 测试3：数据库检查

1. **打开云开发控制台**
   - 进入"数据库"

2. **检查 craftsmen 集合**
   - 找到当前手艺人记录
   - 确认 `avatarUrl` 字段有值（以 `cloud://` 开头）

3. **检查 users 集合**
   - 找到当前用户记录
   - 确认 `avatarUrl` 字段有值

---

## 🔧 常见问题

### 问题1：上传失败
**检查点**：
- 云存储是否有写入权限
- 云函数是否已部署
- 控制台错误日志

### 问题2：头像显示 403
**原因**：使用的是 fileID 而不是临时 URL
**解决**：已在前端代码中自动转换

### 问题3：刷新后头像消失
**检查点**：
- 数据库是否成功保存
- 登录接口是否返回 avatarUrl
