# 注册流程修复总结

## 问题描述
用户反馈的问题：
1. 申请后直接显示"审核通过"
2. 可以切换到管理员角色
3. 正确逻辑应该是：申请后显示"等待审批"，返回登录页，管理员通过后才能登录

## 修复内容

### 1. 删除自动审核通过弹窗
**文件**: `pages/common/pendingApproval.js`

删除了页面加载时自动检测并弹窗的逻辑，改为只显示 Toast 提示。

### 2. 修改注册成功后的跳转
**文件**: 
- `pages/auth/craftsmanRegister.js`
- `pages/auth/dispatcherAuth.js`

修改前：注册后跳转到 `pendingApproval` 页面
修改后：注册后返回登录页 `/pages/login/index`

### 3. 添加返回登录按钮
**文件**: `pages/common/pendingApproval.wxml` 和 `.js`

在审核中状态添加了"返回登录"按钮，让用户可以手动返回登录页。

## 正确的注册流程

```
1. 用户填写注册表单
   ↓
2. 提交注册
   ↓
3. 显示"申请已提交，等待审批"
   ↓
4. 返回登录页面
   ↓
5. 用户尝试登录
   ↓
6. 被拦截，显示"等待审批"提示
   ↓
7. 可以查看审批状态页面
   ↓
8. 管理员审批通过
   ↓
9. 用户重新登录
   ↓
10. 成功进入系统
```

如果测试仍然失败，请检查数据库：

```javascript
// 1. 查看用户状态
const phone = '测试手机号'
const user = await db.collection('users').where({ phone }).get()
console.log('roles:', user.data[0].roles)
console.log('roleApplications:', user.data[0].roleApplications)

// 2. 如果 roles 中已经有该角色，说明是遗畜数据

    roles: [], // 清空

  }
})
```

## 常见问题

### 问题 1: 注册后直接通过
**可能原因**: 数据库中该手机号已经有角色数据
**解决**: 清理数据库中该手机号的数据

### 问题 2: 注册后跳转错误
**可能原因**: 前端缓存了旧版本
**解决**: 清理微信开发者工具缓存，重新编译

### 问题 3: 登录时不拦截
**可能原因**: 云函数未部署成功
**解决**: 重新部署 user 云函数
