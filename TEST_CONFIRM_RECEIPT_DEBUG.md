# 确认收款功能 - 调试与测试指南

## 🎯 快速开始

现在页面已添加调试信息和测试按钮，可以帮助你快速测试确认收款功能。

---

## 📱 界面说明

### 调试信息区域
在订单详情页顶部，你会看到黄色背景的调试信息框：

```
🔧 调试信息
status: completed
paymentStatus: unpaid
canConfirmReceipt: false
craftsmanId: xxx
order.craftsmanId: xxx
```

### 测试按钮（仅在开发模式显示）

根据当前订单状态，会显示不同的测试按钮：

| 当前状态 | 显示的测试按钮 | 功能 |
|---------|--------------|------|
| `completed` + `unpaid` | 🧪 测试：模拟派单人确认付款 | 将 paymentStatus 改为 `pending_receipt` |
| `completed` + `pending_receipt` | 🧪 测试：直接确认收款 | 直接完成收款，订单变为 `closed` + `paid` |

---

## 🧪 测试场景

### 场景 1：测试"确认收款"按钮显示

**当前状态**：订单已完成，等待派单人付款

**步骤**：
1. 手艺人进入订单详情页
2. 查看调试信息区域
3. 确认看到 `status: completed`，`paymentStatus: unpaid`
4. 看到提示"等待派单人确认付款"

**预期结果**：
- 此时**不会**显示"确认收款"按钮（正常，因为派单人还没确认付款）
- 显示橙色提示框"派单人确认付款后，您将可以确认收款"

---

### 场景 2：模拟派单人确认付款

**步骤**：
1. 点击调试区域的按钮 🧪 **测试：模拟派单人确认付款**
2. 确认对话框
3. 等待操作完成

**预期结果**：
- 页面自动刷新
- `paymentStatus` 变为 `pending_receipt`
- 看到绿色按钮 ✅ **确认收款**

---

### 场景 3：测试确认收款功能

**当前状态**：订单已完成，paymentStatus 为 `pending_receipt`

**步骤**：
1. 点击绿色按钮 ✅ **确认收款**
2. 确认对话框"确定已收到款项吗？"
3. 等待操作完成

**预期结果**：
- 提示"收款确认成功"
- 页面刷新后显示"订单已结束"
- `status` 变为 `closed`
- `paymentStatus` 变为 `paid`

---

### 场景 4：快速测试完整流程

如果你想快速测试而不等待派单人操作：

**步骤**：
1. 点击 🧪 **测试：直接确认收款**
2. 确认对话框
3. 等待操作完成

**预期结果**：
- 订单直接变为 `closed` + `paid` 状态
- 显示"订单已结束"

## 🔍 问题排查

### 问题 1：调试信息不显示

**检查**：
```javascript
// pages/craftsman/orderDetail.js
data: {
  // ...
  showDebugInfo: true  // 确保这里是 true
}
```

---

### 问题 2：canConfirmReceipt 为 false

查看调试信息中的字段：

| 字段 | 期望值 | 说明 |
|-----|-------|-----|
| `status` | `completed` | 订单必须已完成 |
| `paymentStatus` | `pending_receipt` | 必须等待派单人确认付款后 |
| `craftsmanId` | 与 `order.craftsmanId` 相同 | 当前用户必须是接单人 |

**如果 `craftsmanId` 不匹配**：
- 说明你当前登录的账号不是该订单的接单人
- 需要切换到手艺人账号，或由实际接单人操作

---

### 问题 3：测试按钮点击无反应

**检查控制台日志**：
1. 打开微信开发者工具控制台
2. 查看是否有报错信息
3. 检查网络请求是否成功

**常见问题**：
- 数据库权限不足 → 确保手艺人有修改订单的权限
- 订单已被删除 → 检查订单ID是否有效

## 🔧 关闭调试模式

测试完成后，关闭调试信息：

```javascript
// pages/craftsman/orderDetail.js
data: {
  // ...
  showDebugInfo: false  // 改为 false
}
```

或者添加长按标题切换功能（已预留接口）：
```javascript
// 在页面中添加
onTitleLongPress() {
  this.toggleDebugInfo()
}
```

## 💡 生产环境部署

上线前请确保：

1. **关闭调试模式**
   ```javascript
   showDebugInfo: false
   ```

2. **移除测试按钮**（可选）
   - 调试信息区域会自动隐藏
   - 或手动删除 WXML 中的调试代码

3. **测试完整真实流程**
   - 使用真实账号测试派单人确认付款 → 手艺人确认收款
