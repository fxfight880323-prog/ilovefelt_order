# 端到端完整测试指南

本文档提供从头到尾的完整测试流程，覆盖用户注册、审批、登录、订单全流程。

## 测试场景概览

| 场景 | 测试内容 |
|------|----------|
| 场景1 | 新用户注册为手艺人 → 管理员审批 → 登录 → 接单/取消/完成 |
| 场景2 | 新用户注册为派单人 → 管理员审批 → 登录 → 创建订单/派单/取消 |
| 场景3 | 同一用户注册双角色 → 管理员审批 → 登录 → 切换角色测试 |

---

## 前置准备

### 1. 确保云函数已部署

在微信开发者工具中，右键以下云函数选择 "创建并部署：云端安装依赖"：
- `cloudfunctions/user`
- `cloudfunctions/admin`
- `cloudfunctions/order`
- `cloudfunctions/craftsman`
- `cloudfunctions/sms` (如需测试真实短信)

### 2. 设置管理员账号

**方法1：通过数据库直接设置**（推荐测试环境使用）

在 `users` 集合中创建一条记录：
```json
{
  "openid": "管理员的openid",
  "phone": "13810062394",
  "name": "管理员",
  "roles": ["admin"],
  "currentRole": "admin",
  "isAdmin": true,
  "status": "active",
  "roleApplications": [
    {
      "role": "admin",
      "status": "active",
      "applyTime": {"$date": "2026-04-01T00:00:00.000Z"}
    }
  ],
  "createTime": {"$date": "2026-04-01T00:00:00.000Z"},
  "updateTime": {"$date": "2026-04-01T00:00:00.000Z"}
}
```

### 3. 数据库集合权限设置

确保以下集合权限已设置为 "所有用户可读，仅创建者可写" 或 "所有用户可读可写"（测试环境）：
- `users`
- `craftsmen`
- `dispatchers`
- `orders`

## 场景1：手艺人完整流程测试

### 步骤1.1：新用户注册为手艺人

**操作路径：** 登录页 → 微信登录 → 未注册提示 → 去注册 → 注册为手艺人

**测试步骤：**
1. 打开小程序，进入登录页面
2. 勾选 "同意用户协议"
3. 点击 "微信一键登录"
4. 系统提示 "未注册"，点击 "去注册"
5. 选择 "注册为手艺人"
6. 填写注册信息：
   - 姓名：测试手艺人A
   - 手机号：13800138001（测试号段，避免与真实用户冲突）
   - 验证码：在控制台查看测试验证码 或 在 `verifyCodes` 集合中查找
   - 擅长工艺：木工
   - 从业经验：3-5年
   - 所在地区：北京市朝阳区
   - 登录密码：123456
   - 确认密码：123456
   - 勾选同意协议
7. 点击 "提交申请"

**预期结果：**
- 显示 "申请已提交" 弹窗
- 提示 "需要管理员审批后才能登录使用"
- 返回登录页面

**数据库验证：**
```javascript
// 在云开发控制台数据库查询
db.collection('users').where({phone: '13800138001'}).get()
// 应有记录，status: 'pending', roles: []

db.collection('craftsmen').where({phone: '13800138001'}).get()
// 应有记录，status: 'pending'
```

---

### 步骤1.2：管理员审批手艺人

**操作路径：** 管理员登录 → 管理控制台 → 用户审批 → 找到申请 → 通过

**测试步骤：**
1. 使用管理员账号登录（手机号：13810062394）
2. 进入 "管理控制台"
3. 找到 "用户审批" 或 "角色申请管理"
4. 找到 "测试手艺人A" 的申请记录
5. 点击 "通过审批"

**预期结果：**
- 显示 "审批通过"
- 申请人收到订阅消息通知（如已配置）

**数据库验证：**
```javascript
db.collection('users').where({phone: '13800138001'}).get()
// 应有记录：
// - roles: ['craftsman']
// - roleApplications[0].status: 'active'

db.collection('craftsmen').where({phone: '13800138001'}).get()
// status 应变为 'active'
```

---

### 步骤1.3：手艺人登录

**操作路径：** 登录页 → 微信登录/手机号登录

**测试步骤：**
1. 打开小程序
2. 使用以下任一方式登录：
   - **微信登录**：点击 "微信一键登录"（自动识别已审批）
   - **手机号登录**：输入 13800138001 / 密码 123456

**预期结果：**
- 登录成功
- 自动跳转到手艺人首页（接单大厅）
- 底部Tab显示：首页、接单大厅、我的订单、个人中心

---

### 步骤1.4：手艺人接单

**前置条件：** 派单人已创建一个待接单订单

**操作路径：** 接单大厅 → 选择订单 → 立即接单

**测试步骤：**
1. 进入 "接单大厅" 页面
2. 查看待接单列表
3. 点击一个订单卡片进入详情
4. 确认订单信息（样式、数量、单价、收货日期）
5. 点击 "立即接单"
6. 确认弹窗点击 "确定"

**预期结果：**
- 显示 "接单成功"
- 订单从待接单列表消失
- 进入 "我的订单" 可看到已接订单

**数据库验证：**
```javascript
db.collection('orders').doc('订单ID').get()
// status: 'accepted'
// craftsmanId: 手艺人ID
// craftsmanName: '测试手艺人A'
```

---

### 步骤1.5：手艺人取消订单

**操作路径：** 我的订单 → 选择进行中的订单 → 取消订单

**测试步骤：**
1. 进入 "我的订单" 页面
2. 找到状态为 "进行中" 的订单
3. 点击进入订单详情
4. 点击 "取消订单" 按钮
5. 填写取消原因（可选）
6. 确认取消

**预期结果：**
- 显示 "订单已取消"
- 订单状态变为 "已取消"
- 订单回到待接单列表（其他手艺人可接）

**数据库验证：**
```javascript
db.collection('orders').doc('订单ID').get()
// status: 'cancelled'
// cancelledBy: 'craftsman'
// cancelTime: 时间戳
```

---

### 步骤1.6：重新接单并完成订单

**完整流程：**
1. 再次接单（重复步骤1.4）
2. 进入订单详情，点击 "填写运单号"
3. 输入运单号和快递公司
4. 点击 "确认发货"
5. 订单状态变为 "已发货"
6. 点击 "上传完成照片"
7. 选择/拍摄成品照片（至少1张）
8. 点击 "确认完成"
9. 等待派单人确认付款
10. 收到付款通知后，点击 "确认收款"
11. 订单完成

**预期结果：**
- 每个步骤都有成功提示
- 最终订单状态：completed
- 手艺人累计完成数 +1

   - 手机号：13800138002

   - 所属公司：测试公司（可选）
   - 登录密码：123456
5. 提交申请

**预期结果：**
- 显示 "申请已提交，等待审批"
- 返回登录页

---

### 步骤2.2：管理员审批派单人

**操作路径：** 同步骤1.2

**测试步骤：**
1. 管理员登录
2. 进入用户审批
3. 找到 "测试派单人B"

**预期结果：**
- 审批通过
- 用户 roles 添加 'dispatcher'

### 步骤2.3：派单人登录

**测试步骤：**
1. 使用 13800138002 / 123456 登录

**预期结果：**
- 登录成功
- 进入派单人首页

---

### 步骤2.4：派单人创建订单

**操作路径：** 首页 → 新建订单 → 填写信息 → 提交

**测试步骤：**
1. 点击 "新建订单" 或 "+" 按钮
2. 填写订单信息：
   - 订单名称：测试订单001
   - 样式：选择已有样式或新建
   - 数量：10
   - 单价：50
   - 收货日期：3天后
   - 备注：测试订单，请按时完成
3. 点击 "创建订单"

**预期结果：**
- 显示 "创建成功"
- 生成10位订单编号（如 AB24031578）
- 订单出现在 "我的订单" 列表
- 手艺人收到新订单通知（如配置订阅消息）

**数据库验证：**
```javascript
db.collection('orders').where({name: '测试订单001'}).get()
// status: 'pending'
// dispatcherId: 派单人ID
// orderCode: 10位编码
```

---

### 步骤2.5：派单人派单给指定手艺人

**说明：** 当前系统采用 "手艺人抢单" 模式，如需指定派单，需要管理员在后台操作。

**替代方案 - 推荐手艺人：**
1. 创建订单时，在备注中指定推荐手艺人
2. 通过其他渠道（微信）通知该手艺人

---

### 步骤2.6：派单人取消订单

**操作路径：** 我的订单 → 选择待接单订单 → 取消订单

**测试步骤：**
1. 进入 "我的订单"
2. 找到状态为 "待接单" 的订单
3. 点击 "取消订单"
4. 填写原因（可选）
5. 确认

**预期结果：**
- 订单状态变为 cancelled
- 从待接单列表移除

**注意：** 已接单的订单无法直接取消，需要联系手艺人或管理员处理。

---

### 步骤2.7：派单人确认付款

**前置条件：** 手艺人已完成订单并上传照片

**操作路径：** 我的订单 → 已发货订单 → 确认付款

**测试步骤：**
1. 找到状态为 "已发货" 的订单
2. 点击查看详情
3. 查看手艺人上传的完成照片
4. 确认无误后点击 "确认付款"
5. 选择付款方式（银行转账/微信/支付宝）
6. 填写备注
7. 确认

**预期结果：**
- 订单状态：paymentStatus = 'pending_receipt'
- 手艺人收到付款通知

## 异常场景测试

### 场景A：未审批用户登录

**测试步骤：**
1. 注册新账号（不要审批）
2. 尝试登录

**预期结果：**
- 提示 "您的账号正在审核中，请等待管理员审批"
- 提供 "查看状态" 按钮跳转到等待页面

### 场景B：审批被拒绝后重新申请

**测试步骤：**
1. 管理员拒绝用户的申请
2. 用户登录查看状态
3. 点击 "重新申请"
4. 修改信息后再次提交

**预期结果：**
- 可以重新提交申请
- 旧申请记录被覆盖或标记为重新申请

### 场景C：手艺人重复接单

**测试步骤：**
1. 手艺人A接单成功
2. 手艺人B尝试接同一单

**预期结果：**
- 手艺人B提示 "该订单已被接"

### 场景D：无权限操作

**测试步骤：**
1. 手艺人尝试创建订单
2. 派单人尝试接单

**预期结果：**
- 提示 "无权限操作"

## 常见问题排查

### Q1：注册后无法登录，提示 "正在审核中"
**解决：** 这是正常流程，需要管理员在后台审批。

### Q2：审批通过后仍显示审核中
**解决：** 
1. 检查 users 集合中 roleApplications 的 status 是否为 'active'
2. 检查 roles 数组是否包含对应角色
3. 清除小程序缓存重新登录

### Q3：手艺人看不到订单
**解决：**
1. 检查 orders 集合中 status 是否为 'pending'
2. 检查手艺人 craftsman 记录 status 是否为 'active'

### Q4：接单时提示 "您还未注册成为手艺人"
**解决：**
1. 检查 globalData.craftsmanInfo 是否正确获取
2. 检查 craftsmen 集合中是否有对应 openid 的记录

### Q5：派单人创建订单失败
**解决：**
1. 检查 dispatchers 集合中是否有对应记录
2. 检查用户 role 是否为 'dispatcher'
