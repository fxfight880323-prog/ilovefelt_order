# 新订单订阅消息测试指南

## 📋 测试前准备

### 1. 确认模板ID已配置
文件：`config/subscribe.js`
```javascript
NEW_ORDER: 'dUvWdLqKFxuJmYM69-V0X7SJ0LfeBx5bS9igrDgf6-4'
```

### 2. 确认模板字段匹配
请登录微信公众平台检查模板字段，确保与代码中的字段对应：

代码中使用的字段：
- `thing1` - 订单名称
- `thing2` - 样式
- `number3` - 数量
- `amount4` - 总价
- `date5` - 收货日期

如果模板字段不同，需要修改 `cloudfunctions/message/index.js` 中的字段名。

---

## 🧪 测试步骤

### 步骤1：手艺人订阅新订单提醒

1. **手艺人进入接单大厅**
   - 打开小程序，切换到手艺人角色
   - 进入"接单大厅"页面

2. **触发订阅授权**
   - 页面会弹出提示："开启消息提醒"
   - 点击"去开启"
   - 勾选"新订单提醒"，点击"允许"

3. **验证订阅保存**
   - 订阅成功后，数据库 `subscribers` 集合会新增记录
   - 可以在云开发控制台查看

### 步骤2：派单人创建新订单

1. **切换到派单人角色**
   - 进入"发布订单"页面

2. **创建新订单**
   - 填写订单信息
   - 点击"提交"

3. **验证消息发送**
   - 订单创建后，系统会自动调用 `sendNewOrderNotice`
   - 已订阅的手艺人应该收到微信订阅消息

### 步骤3：验证订阅消息接收

1. **手艺人收到消息**
   - 微信服务通知会显示新订单提醒
   - 点击消息可跳转到接单大厅

## 🔧 常见问题排查

### 问题1：没有弹出订阅授权
- 检查模板ID是否为空
- 检查是否已经授权过（只能授权一次）
- 在微信开发者工具中清除缓存重试

### 问题2：订阅成功但没收到消息
- 检查云函数 `message` 是否部署
- 检查数据库 `subscribers` 是否有记录
- 查看云函数调用日志是否有错误
- 确认模板ID正确且已通过审核

### 问题3：消息发送失败
- 检查模板字段是否与代码匹配
- 查看 `cloudfunctions/message` 日志
- 确认用户没有取消订阅

---

## 📱 重新测试方法

如果测试失败，需要重置订阅状态：

1. **删除订阅记录**
   ```javascript
   // 在云开发控制台数据库中执行
   db.collection('subscribers').remove()
   ```

2. **清除小程序缓存**
   - 微信开发者工具：工具 → 清除缓存 → 全部清除
   - 或删除小程序重新进入

3. **重新编译测试**
