# 订单图片显示功能测试指南

## 📋 测试前准备

1. **确保有带图片的订单**
   - 如果还没有，需要先创建带图片的订单

2. **重新编译小程序**
   ```
   按 Ctrl + R 重新编译
   ```

---

## 🧪 测试步骤

### 测试1：创建带图片的订单

1. **进入派单人页面**
   - 切换到派单人角色
   - 点击"发布订单"

2. **填写订单信息并上传图片**
   - 选择样式、填写数量价格
   - 点击"上传图片"（如果有这个功能）
   - 提交订单

3. **验证图片上传成功**
   - 在云开发控制台 → 云存储中查看是否有图片
   - 数据库 orders 表中该订单的 imageUrl 字段应有值

### 测试2：在订单管理中查看图片

1. **进入管理员后台**
   - 切换到管理员角色
   - 进入"订单管理"

2. **查看订单列表**
   - 找到刚才创建的带图片订单
   - **预期结果**：
     - 订单信息下方显示订单图片
     - 图片显示为 200x200 的缩略图
     - 图片下方有"点击查看大图"提示

3. **点击图片预览**
   - 点击订单图片
   - **预期结果**：
     - 进入全屏图片预览模式
     - 可以左右滑动查看其他订单图片
     - 点击空白处退出预览

### 测试3：不同状态的订单图片显示

1. **待接单订单**
   - 图片应正常显示

2. **进行中订单**
   - 图片应正常显示

3. **已完成订单**
   - 图片应正常显示

---

## 🔧 常见问题

### 问题1：图片不显示
**检查点**：
- 数据库中该订单的 imageUrl 字段是否有值
- 图片是否为有效的 fileID 或 URL
- 控制台是否有 403 错误

### 问题2：图片显示 403
**原因**：fileID 没有转换为临时 URL
**解决**：代码已自动转换，检查是否重新编译

### 问题3：预览功能无效
**检查点**：
- 点击事件是否正确绑定
- imageUrl 是否正确传递到 data-url
