# 管理员审批功能实现指南

## 功能概述

本实现完成了以下功能：

## 数据流说明

### 注册流程

```
用户提交注册
    ↓
数据写入craftsmen/dispatchers表，status=pending
    ↓
提示"等待管理员审核"
    ↓
管理员在控制台看到待审核申请
    ↓
管理员审批通过/拒绝
    ↓
用户状态变为active，可正常使用
```

### 审批流程

```
管理员进入"待审核"标签
    ↓
查看手艺人和派单人申请列表
    ↓
点击"通过"或"拒绝"
    ↓
如通过：更新状态为active，用户获得角色权限
    ↓
如拒绝：更新状态为rejected，可填写拒绝原因
```

## 注意事项

1. **旧数据兼容**：已有订单和角色不受影响
2. **手机号唯一性**：一个手机号只能注册一个角色类型
3. **管理员特权**：管理员手机号可直接通过审核
4. **审批通知**：后续可增加审批结果通知功能

## 后续优化建议

1. **审批通知**：审核结果通过微信消息通知用户
2. **批量审批**：支持批量通过或拒绝申请
3. **审批历史**：查看已处理的审批记录
4. **黑名单**：拒绝的用户可加入黑名单
