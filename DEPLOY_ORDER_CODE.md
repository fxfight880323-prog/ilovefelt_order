# 订单唯一编码 - 部署和测试指南

## 📦 修改内容

### 云函数修改 (`cloudfunctions/order/index.js`)

1. **新增 `generateOrderCode` 函数**
   - 生成10位唯一编码（1位字母 + 9位数字）
   - 格式：`A240315078`
   - 包含日期信息和随机数
   - 检查数据库确保唯一性

2. **修改 `createOrder` 函数**
   - 调用 `generateOrderCode` 生成编码
   - 将 `orderCode` 写入数据库
   - 返回 `orderCode` 给前端

### 前端修改

#### 手艺人端
- `pages/craftsman/orderDetail.wxml`：显示 `#ORDER-CODE`

## 📋 编码格式验证

创建订单后，检查编码是否符合以下格式：

```
格式：A240315078
      ││││││└─┘
      │││││└──┴─ 随机数（3位：000-999）
      │││└─┴──── 日期（2位：01-31）
      │└─┴────── 月份（2位：01-12）
      └───────── 年份后2位（如 24）
                 字母（A-Z，排除 I、O）
```

**示例验证**：
- ✅ `A240315078` - 正确
- ✅ `K241201003` - 正确
- ❌ `I240315078` - 错误（包含 I）
- ❌ `A24031578` - 错误（只有9位）
- ❌ `A2403150789` - 错误（11位）

## 🔍 故障排查

### 问题 1：订单编码未生成

**现象**：创建订单后，数据库中没有 `orderCode` 字段

**排查**：
1. 检查云函数是否已部署
2. 检查控制台是否有错误日志
3. 检查 `generateOrderCode` 函数是否被调用

### 问题 2：编码重复冲突

**现象**：创建订单时返回"无法生成唯一订单编码"

**原因**：当天生成的订单过多，随机数冲突

**解决**：
- 稍等片刻后重试（随机数会重新生成）
- 检查是否有异常的大量订单创建

### 问题 3：前端不显示编码

**现象**：订单详情页显示 `#undefined` 或长串ID

**排查**：
1. 检查订单是否为新创建（旧订单没有 orderCode）
2. 检查云函数返回数据是否包含 orderCode
3. 检查 WXML 代码是否正确：`{{orderInfo.orderCode || orderInfo._id}}`

### 2. 按编码查询订单

添加按订单编码查询的功能：

```javascript
// 云函数中添加查询接口
async function getOrderByCode(params) {
  const { orderCode } = params
  const order = await db.collection('orders')
    .where({ orderCode })
    .get()
  return order.data[0]
}
```
