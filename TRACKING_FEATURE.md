# 运单号功能说明

## 功能概述

已实现运单号管理功能：
- ✅ 匠人完成订单时填写运单号
- ✅ 支持选择物流公司
- ✅ 派单人可查看运单信息
- ✅ 支持复制运单号

## 订单状态流转

```
待接单 (pending)
    ↓ 匠人接单
进行中 (accepted)
    ↓ 匠人完成并填写运单号
已发货 (shipped) ← 新增状态
    ↓ 买家确认收货
已完成 (completed)
```

## 数据库字段

订单集合新增字段：
```javascript
{
  trackingNumber: '',    // 运单号
  trackingCompany: '',   // 物流公司
  trackingStatus: '',    // 物流状态: none/pending/shipped/delivered
  shipDate: '',          // 发货时间
}
```

## 使用流程

### 匠人端

1. **接单**
   - 进入"接单大厅"
   - 点击订单"立即接单"

2. **完成订单并填写运单号**
   - 在"我的订单"中找到订单
   - 点击"完成并填写运单号"
   - 选择物流公司（默认顺丰）
   - 粘贴运单号
   - 点击"提交运单号"
   - 订单状态变为"已发货"

3. **查看运单信息**
   - 订单详情显示运单号和物流公司
   - 可复制运单号

### 派单人端

1. **查看运单信息**
   - 进入"订单管理"
   - 找到已发货的订单
   - 可看到运单号和物流公司
   - 可复制运单号查询物流

2. **筛选已发货订单**
   - 在订单列表筛选"已发货"状态

## 界面说明

### 订单状态显示
- 🔄 进行中 → 可点击"完成并填写运单号"
- 🚚 已发货 → 显示运单信息
- ✅ 已完成 → 订单完结

### 运单号输入
- 支持选择主流物流公司
- 运单号必填
- 提交前确认提示

## 云函数接口

### 添加运单号
```javascript
wx.cloud.callFunction({
  name: 'order',
  data: {
    action: 'addTracking',
    orderId: '订单ID',
    trackingNumber: 'SF123456789',
    trackingCompany: '顺丰速运',
    craftsmanId: '匠人ID'
  }
})
```

## 注意事项

1. **权限控制**
   - 只有接单匠人可填写运单号
   - 其他人无法修改运单信息

2. **状态变更**
   - 填写运单号后订单状态变为"已发货"
   - 已发货订单不能再修改运单号（如需要修改，联系管理员）

3. **物流公司**
   - 支持：顺丰、中通、圆通、申通、韵达、EMS、京东、其他
   - 默认选中"顺丰速运"
